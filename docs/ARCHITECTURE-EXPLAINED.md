# Архитектура проекта: Faust.js - Объяснение

## Что такое Faust.js и FaustWP?

**Faust.js** и **FaustWP** - это инструменты, которые связывают WordPress и Next.js вместе. Это "мост" между headless WordPress и Next.js frontend.

### Аналогия
Представьте:
- **WordPress** = склад с товарами (контент)
- **Next.js** = магазин, где показывают товары (фронтенд)
- **Faust.js/FaustWP** = транспортная система, которая доставляет товары со склада в магазин

## Компоненты Faust

### 1. FaustWP (плагин в WordPress)
- Устанавливается в WordPress
- Дает механизмы для:
  - **Preview** - когда редактор нажимает "Предпросмотр" в WordPress, он видит реальный Next.js сайт
  - **Аутентификация** - безопасная связь между WordPress и Next.js
  - **Интеграция** - точки подключения для обмена данными

### 2. Faust.js (библиотека в Next.js)
- Устанавливается в Next.js проект (`@faustwp/core`, `@faustwp/cli`)
- Предоставляет:
  - Хелперы для получения данных из WordPress
  - Роутинг, соответствующий WordPress структуре
  - Поддержку SSG/SSR
  - Preview функциональность

## Preview функциональность - как это работает?

### Без Faust (проблема):
1. Редактор создает пост в WordPress
2. Нажимает "Предпросмотр"
3. Видит только JSON данные или старую версию
4. Не видит, как это будет выглядеть на реальном Next.js сайте

### С Faust (решение):
1. Редактор создает пост в WordPress
2. Нажимает "Предпросмотр" в WordPress админке
3. **FaustWP** создает временную ссылку с preview токеном
4. WordPress перенаправляет на Next.js сайт
5. **Faust.js** в Next.js получает preview токен
6. **Faust.js** запрашивает черновик поста через GraphQL с preview токеном
7. Next.js показывает preview версию поста
8. Редактор видит, как пост будет выглядеть на реальном сайте

### В коде это выглядит так:

**В Next.js (Faust.js):**
```typescript
// Автоматически обрабатывает preview запросы
import { getWordPressProps } from '@faustwp/core';

export default function Page(props) {
  // props уже содержат данные из WordPress
  // Если это preview - данные черновика
  return <div>{props.post.title}</div>;
}
```

**В WordPress (FaustWP):**
- Плагин автоматически добавляет кнопку "Preview" в редакторе
- При нажатии создает preview URL с токеном
- Перенаправляет на Next.js сайт

## Примеры использования

### Локальная разработка:
```
WordPress (localhost:8002) ←→ Next.js (localhost:3000)
         FaustWP + Faust.js обеспечивают связь
```

### Production:
```
WordPress (yourdomain.com) ←→ Next.js (на любом хостинге)
         FaustWP + Faust.js обеспечивают связь
```

## Резюме

| Компонент | Что это? | Зачем нужен? |
|-----------|---------|--------------|
| **FaustWP** | Плагин WordPress | Preview, аутентификация, интеграция |
| **Faust.js** | Библиотека Next.js | Получение данных, preview, роутинг |
| **WPGraphQL** | Плагин WordPress | GraphQL API для контента |

**Простыми словами:**
- **Faust** = мост между WordPress и Next.js
- Позволяет редакторам видеть preview в реальном времени
- Упрощает получение данных из WordPress в Next.js
