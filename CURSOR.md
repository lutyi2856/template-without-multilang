## 🛠️ Настройка Cursor

### 1. Правила Cursor (`.cursorrules` или `CURSOR.md`)

- Держите правила **краткими** (до 500 строк)
- Будьте **конкретными** с примерами
- Фокусируйтесь на стандартах кодирования и лучших практиках
- Обновляйте правила по мере развития проекта

### 2. Файл `.cursorignore`

- Исключайте ненужные или чувствительные файлы из индексации
- Предотвращайте загрузку нерелевантных файлов в контекст
- Примеры: `node_modules/`, `.git/`, `dist/`, `build/`, `.env.local`

### 3. Privacy Mode

- Включите Privacy Mode для защиты конфиденциальных данных
- Предотвращает использование вашего кода для обучения моделей других пользователей

### 4. Project Notes (Notepads)

- Ведите детальные заметки о целях и спецификациях проекта
- Используйте Notepads для управления повторяющимися задачами
- Включайте заметки в контекст Composer для лучшего понимания проекта

### 5. Оптимальные настройки

**Готовые настройки:** [CURSOR_OPTIMAL_SETTINGS.json](CURSOR_OPTIMAL_SETTINGS.json)

**Ключевые настройки:**

- **AI Model:** Claude Sonnet 4.5 (Temperature: 0.3)
- **Context:** 100,000 lines, gitignored исключены
- **Performance:** Memory 4096 MB, Background Agent ON
- **Linting:** Auto-fix ON, Iterate on Lints ON
- **Privacy:** Privacy Mode ON, Telemetry OFF

**Как применить:**

1. Открыть [CURSOR_OPTIMAL_SETTINGS.json](CURSOR_OPTIMAL_SETTINGS.json)
2. Скопировать нужные настройки
3. Вставить в Cursor Settings (JSON)
4. Перезапустить Cursor

## 🎯 Skills - Генерация правил из чатов

### Команда `/Generate Cursor Rules`

После решения задачи или обсуждения паттернов:

1. Введите `/Generate Cursor Rules` в чате
2. Cursor проанализирует диалог
3. Сгенерирует правила автоматически
4. Сохранит в `.cursor/rules/`

### Когда использовать:

- ✅ После внедрения новой архитектуры
- ✅ После обсуждения best practices
- ✅ После решения сложной проблемы
- ✅ Когда появились новые соглашения

**Пример:**

```
Чат: "Как правильно структурировать компоненты?"
AI: [объяснение best practices]
→ /Generate Cursor Rules
→ Создается правило в .cursor/rules/
```

## ⚡ Slash-команды Cursor

### Основные команды:

| Команда                        | Когда использовать               |
| ------------------------------ | -------------------------------- |
| `/Generate Cursor Rules`       | После обсуждения паттернов       |
| `/Reset Context`               | Контекст перегружен (>50 файлов) |
| `/Add Open Files to Context`   | Нужны все открытые файлы         |
| `/Add Active Files to Context` | Нужны только видимые файлы       |
| `/Disable Iterate on Lints`    | Автоисправление создает проблемы |

### Best Practice:

1. Регулярно используй `/Reset Context` (каждые 10-15 файлов)
2. Добавляй файлы точечно через `@filename`
3. Генерируй правила после инсайтов

## 📦 ПРАВИЛО #1: Переиспользование компонентов

### КРИТИЧНО: ВСЕГДА проверяй список ПЕРЕД созданием нового компонента!

**Правило:**

1. ✅ СНАЧАЛА: Открыть `docs/COMPONENTS-REUSE.md`
2. ✅ Проверить существующие компоненты
3. ✅ Переиспользовать с props
4. ❌ НЕ создавать копии компонентов с другими названиями

**Примеры:**

❌ **ПЛОХО:** Создать `FooterSocialLinks` (копия `SocialLinks`)
✅ **ХОРОШО:** Использовать `<SocialLinks links={footerLinks} />`

❌ **ПЛОХО:** Создать `FooterLogo` (копия `Logo`)  
✅ **ХОРОШО:** Использовать `<Logo />`

❌ **ПЛОХО:** Создать `<h2>Title</h2>` напрямую
✅ **ХОРОШО:** Использовать `<Heading level={2}>Title</Heading>`

