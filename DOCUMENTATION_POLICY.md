# Политика документации проекта

> Дата: 2025-12-30
> Цель: Минимизация избыточной документации

## 🎯 Проблема

Раньше создавалось много временных файлов:

- `IMPROVEMENTS_SUMMARY.md`, `IMPROVEMENTS_COMPLETE.md`
- `CURSOR_SKILLS_GUIDE.md`, `CURSOR_SLASH_COMMANDS.md`
- `README.md` в каждой папке компонентов
- И другие временные файлы

**Результат:** Потеря токенов, времени, сложность навигации.

## ✅ Решение

### 1. Строгое правило для AI

Создано правило `.cursor/rules/no-unnecessary-docs.mdc`:

- ❌ Запрещает создавать временные файлы
- ❌ Запрещает README в каждой папке
- ✅ Разрешает только критичные файлы
- ✅ Требует явного запроса пользователя

### 2. Автоматическое исключение через .gitignore

Добавлены паттерны в `.gitignore`:

```
*_SUMMARY.md
*_COMPLETE.md
*_CHECKLIST.md
*_GUIDE.md
*_WORKFLOW.md
ANSWER_TO_*.md
QUICK_START_*.md
RULES_*.md
HYBRID_*.md
PROJECT_*.md
```

**Эффект:** Даже если файл создан, он не попадет в git.

### 3. Исключение из контекста через .cursorignore

Добавлены те же паттерны в `.cursorignore`:

- Временные файлы не попадают в контекст AI
- README в папках компонентов игнорируются
- Экономия токенов

## 💡 Что делать вместо создания файлов

### Вместо SUMMARY.md:

```
✅ Написать ответ в чате
❌ НЕ создавать файл
```

### Вместо README.md в компоненте:

```
✅ Добавить JSDoc комментарии в код
❌ НЕ создавать README.md
```

### Вместо CHECKLIST.md:

```
✅ Использовать todo_write
❌ НЕ создавать файл
```

## 🔍 Как проверить работу

### 1. Попросить AI создать временный файл:

```
User: "Создай SUMMARY.md с описанием изменений"
AI: "Я не могу создать SUMMARY.md (правило no-unnecessary-docs.mdc).
     Вместо этого я опишу изменения в чате..."
```

### 2. Проверить .gitignore:

```bash
cat .gitignore | grep "_SUMMARY"
# Должно вывести: *_SUMMARY.md
```

### 3. Проверить .cursorignore:

```bash
cat .cursorignore | grep "_SUMMARY"
# Должно вывести: *_SUMMARY.md
```

## 🔄 Если нужно создать новый документ

### Спросить себя:

1. **Пользователь явно попросил?**

   - НЕТ → ❌ Не создавать
   - ДА → Идти дальше

2. **Файл уже существует?**

   - ДА → ✅ Обновить существующий
   - НЕТ → Идти дальше

3. **Можно добавить в существующий?**

   - ДА → ✅ Добавить в существующий
   - НЕТ → Идти дальше

4. **Это временная информация?**

   - ДА → ❌ Написать в чате
   - НЕТ → Идти дальше

5. **Файл будет использоваться >1 месяца?**
   - НЕТ → ❌ Не создавать
   - ДА → ✅ Спросить пользователя

1. Добавить в `.cursor/rules/no-unnecessary-docs.mdc`
2. Добавить в `.gitignore`
3. Добавить в `.cursorignore`
4. Обновить этот файл
